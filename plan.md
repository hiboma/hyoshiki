# 交通標識読み上げアプリ - 開発計画

## 概要

日本の交通標識を子供向けに教えるブラウザアプリを作成します。
国土交通省の道路標識一覧 PDF からデータを抽出し、静的サイトとして構成します。

## フェーズ

### Phase 1: プロトタイプ (完了)

サンプル標識 16 個でアプリの動作を検証しました。

- [x] PDF からサンプル標識画像を切り出す
- [x] サンプルデータの JSON を作成する
- [x] HTML/CSS/JS で単一ページアプリを構築する
- [x] Web Speech API による読み上げ動作を検証する
- [x] スワイプ操作の動作を検証する
- [x] 設定画面 (音声選択・速度・高さ) を実装する
- [x] 画像処理パイプラインを確立する

### Phase 2: データ整備 (完了)

PDF 全体から標識データを抽出しました。

- [x] 全標識のセル座標を定義する
- [x] 画像処理パイプラインで全標識を一括抽出する
- [x] 標識番号・名称・カテゴリの JSON データを整備する (189 件)
- [x] ふりがな (reading) を全標識に追加する
- [ ] 補助標識 (31 件) の画像を整備する (技術的制約により保留)

### Phase 3: UI 改善 (完了)

子供向けの使いやすさを向上させました。

- [x] カテゴリ別フィルタリング (グリッドページ)
- [x] ルビ (ふりがな) 表示 (カテゴリナビ・カード表示)
- [x] カードページにカテゴリラベルを表示する
- [x] タップフィードバック (スケールアニメーション + ハプティクス)

### Phase 4: 配布・拡張 (未着手)

- [ ] iOS Safari / Android Chrome で動作確認する
- [ ] オフライン対応 (Service Worker)
- [ ] PWA 化 (ホーム画面追加)
- [ ] 補助標識の画像を別手段で整備する

## ファイル構成

```
kotsuh/
  index.html              # アプリ本体
  style.css               # スタイルシート
  app.js                  # アプリケーションロジック
  plan.md                 # 開発計画 (本ファイル)
  data/
    signs.json            # 標識データ (189 件)
    signs/                # 標識画像 (透過 PNG, 220 ファイル)
  docs/
    specification.md      # 仕様書
  scripts/
    extract_signs.py      # PDF から標識画像を抽出するスクリプト
  sources/
    ichiran.pdf           # ソース PDF (国土交通省)
    README.md             # 素材の取得元情報
```

## ソースデータ

- PDF: https://www.mlit.go.jp/road/sign/sign/douro/ichiran.pdf
- ローカルコピー: `sources/ichiran.pdf`
