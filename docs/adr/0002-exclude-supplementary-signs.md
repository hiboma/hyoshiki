# ADR-0002: 補助標識の除外

## ステータス

採用

## コンテキスト

PDF には補助標識 (500 番台) が 31 件含まれていますが、
画像抽出パイプライン (ADR-0001) で正常に切り出せません。

## 問題の詳細

### 1. 白背景に薄い灰色枠線

補助標識の多くは白背景に薄い灰色の枠線で構成されています。
WHITE_THRESHOLD=240 では枠線ごと透過になり、標識が消えます。
閾値を 230 に下げると枠線は残りますが、テキストラベルも残ります。

### 2. stacked cell

補助標識エリアでは 1 セル内に 2〜3 個の標識が縦に詰め込まれています。
セルを分割すると各標識の画像領域が非常に小さく (高さ 20〜30pt)、
ラベルテキストとの分離が困難です。

### 3. connected component の限界

白背景標識の「edge flood fill」方式ではラベルテキストも白背景上にあるため除去できません。
connected component でテキストを除去しようとすると、標識本体の文字部分も除去されます。

## 決定

補助標識 31 件を signs.json から除外し、189 件 (案内 76 + 警戒 27 + 規制 70 + 指示 16) で運用します。

## 今後の対応方針

以下のいずれかで補助標識を追加することを検討します。

- (A) PDF の補助標識エリアの座標を個別に手動調整して再抽出する
- (B) SVG やイラストで補助標識を手動作成する
- (C) 別のソース画像から取得する
