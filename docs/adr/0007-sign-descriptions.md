# ADR-0007: 標識の説明データと表示方法

## ステータス

採用

## コンテキスト

子供が標識の意味や役割を理解するために、各標識の説明文が必要です。
189 件の標識それぞれに、子供向けの説明・目的・注意点を記述します。

## 決定

### データ構造

`signs.json` に `description` フィールドを追加します。

```json
{
  "id": "101",
  "name": "市町村",
  "description": "いまいるまちのなまえをおしえてくれるひょうしきです。まちにはいったときや、でたときにみつけてみよう。"
}
```

説明文は全てひらがなで記述します (3 歳の子供でも読める)。
1 件あたり 2〜4 文、50〜120 文字を目安にします。

### 説明文の構成

各標識の説明は以下の 3 要素で構成します。

1. **これはなに**: 標識が何を伝えているかを 1 文で説明します
2. **どんなとき**: どこで・どんな場面で見かけるかを説明します
3. **きをつけること**: 子供が知っておくべき注意点を記述します (該当する場合のみ)

### UI 表示方法

カードページに「おしえて」ボタンを追加し、タップするとボトムシートで説明を表示します。

#### ボトムシートを選んだ理由

- 説明文は 2〜4 文あるため、カード画面に常時表示すると窮屈になります
- 子供が興味を持ったときだけ表示する方が、能動的な学習を促します
- ボトムシートはスマートフォンの操作に自然に馴染みます
- 閉じるボタンまたは背景タップで閉じられます

#### 代替案として検討したもの

- **カード下部に常時表示**: カードが縦に長くなりすぎます
- **別ページに遷移**: 画面遷移が増えて操作が複雑になります
- **ツールチップ/ポップオーバー**: モバイルでの操作が不自然です
- **カードの裏面 (フリップ)**: 画像が見えなくなり、説明と標識を同時に確認できません

## 影響

- `signs.json` に `description` フィールド (任意) が追加されます
- `index.html` にボトムシートの HTML 要素が追加されます
- `style.css` にボトムシートのスタイルが追加されます
- `app.js` にボトムシートの開閉ロジックが追加されます
- 既存のテストには影響しません
