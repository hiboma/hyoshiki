# ADR-0001: PDF からの標識画像抽出パイプライン

## ステータス

採用

## コンテキスト

国土交通省の道路標識一覧 PDF から標識画像を取得する必要があります。
PDF は A3 横の単一ページで、Illustrator ベクターで作成されており、標識が表形式で並んでいます。

## 決定

PyMuPDF で PDF を 4 倍解像度 (4763x3368px) でラスタライズし、
セル座標に基づいて個別の標識を切り出します。

### 処理手順

1. PDF を PyMuPDF で PNG にレンダリングします (SCALE=4)
2. 事前定義したセル座標で各標識を crop します
3. 白背景 (RGB >= 240) を透過にします
4. OpenCV の connectedComponentsWithStats で連結成分を解析します
5. 画像中央に最も近い大きな成分を標識本体と判定します
6. 本体の bounding box 外の成分 (ラベルテキスト・罫線) を除去します
7. 本体内部の透過ピクセルを flood fill で判定し、白色に復元します

### 特殊対応

- **白背景標識** (101, 114の2-A, 119-A): 「edge flood fill」方式で外周の白のみを除去します
- **stacked cell** (316/317, 328/328の2 等): 同一画像を複数の標識 ID で共有します

## 代替案

### A. PDF のベクターデータを直接抽出する

PDF 内の描画命令を解析して SVG に変換する方法です。
Illustrator 由来の複雑なパス構造のため、解析が困難であり不採用としました。

### B. Web 上の標識画像を取得する

別のソース (ブログ記事等) から画像を取得する方法です。
検証の結果、PDF からの抽出の方が品質と網羅性が高いため不採用としました。
