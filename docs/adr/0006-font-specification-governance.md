# ADR-0006: フォント仕様のガバナンス

## ステータス

採用

## コンテキスト

開発中にフォントサイズの意図しない変更 (リグレッション) が発生しました。
子供向けアプリでは文字の視認性が重要であり、フォント仕様の変更には注意が必要です。

## 決定

### フォント仕様の変更には承認を必須とします

フォントサイズ・フォントウェイトを変更する場合は、以下の手順に従います。

1. 変更内容をユーザーに提示し、承認を得ます
2. CSS を変更します
3. `tests/app.spec.js` の `FONT_SPECS` の期待値を更新します
4. テストを実行して全てパスすることを確認します

### Playwright テストで仕様を検証します

`tests/app.spec.js` にフォント仕様を `FONT_SPECS` として一元管理し、
CSS の computed style と照合するテストを実行します。

### 現在のフォント仕様

| 要素 | font-size | font-weight |
|---|---|---|
| 標識名 (`#sign-name-block`) | 28px | 700 |
| ふりがな (`#sign-name-block rt`) | 16px | - |
| カテゴリラベル (`#card-category-label span`) | 16px | 600 |
| カウンター (`#counter`) | 14px | - |
| カテゴリボタン (`.category-btn`) | 14px | 600 |
| カテゴリボタンのルビ (`.category-btn rt`) | 9px | - |
| 戻るボタン (`#back-btn`) | 15px | 600 |

## 影響

- `CLAUDE.md` にフォント変更ルールを記載しています。
- コードを変更した後は `npx playwright test` でリグレッションを検出できます。
