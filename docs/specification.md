# 仕様書: 交通標識読み上げアプリ (kotsuh)

## 目的

子供が日本の交通標識の名前と見た目を覚えるための学習アプリです。

## 対象デバイス

- スマートフォン (主要ターゲット、~480px)
- iPad / iPad Air (768px~、5列グリッド)
- iPad Pro / iPad 横向き (1024px~、6列グリッド)
- デスクトップブラウザ (副次的)

## 対象ブラウザ

- iOS Safari 15+
- Android Chrome 90+

## 機能仕様

### F-01: グリッド一覧

- 起動時に全標識をグリッド形式で表示します。列数はビューポート幅に応じて変化します (スマートフォン: 4列、iPad: 5列、iPad Pro: 6列)。
- カテゴリナビゲーションで表示する標識をフィルタリングできます。
- カテゴリは「ぜんぶ」「案内」「警戒」「規制」「指示」の 5 つです。
- カテゴリ名にはルビ (ふりがな) を表示します。
- 標識をタップするとカード表示 (F-02) に遷移します。

### F-02: カード表示

- 画面に 1 つの標識画像を大きく表示します。
- 標識名を漢字とルビ (ふりがな) で表示します。
- 所属カテゴリ名をルビ付きで表示します。
- 現在位置 / 全体数を表示します (例: 3 / 189)。
- 標識画像をタップすると読み上げ (F-03) を行います。

### F-03: 読み上げ

- 標識画像またはテキストをタップすると、ふりがな (reading) を日本語で読み上げます。
- Web Speech API (SpeechSynthesis) を使用します。
- `lang: "ja-JP"` を指定します。
- iOS Safari ではユーザー操作起因でのみ発話が可能です。タップイベントから呼び出すため問題ありません。

### F-04: ナビゲーション

- 左右スワイプで前後の標識に切り替えます。
- Touch Events API (`touchstart` / `touchend`) で方向を判定します。
- 左右ボタンも設置します (スワイプが難しい場合の代替)。
- 最初/最後の標識でループします。
- スライドアニメーション付きで切り替えます。
- デスクトップでは左右矢印キー、スペース/Enter キーにも対応します。

### F-05: 設定

- 歯車アイコンから設定画面を開きます。
- 以下の項目を設定できます。
  - 音声の選択 (端末にインストールされた日本語音声の一覧から選択)
  - 読み上げ速度 (0.5〜1.5、デフォルト: 0.9)
  - 声の高さ (0.5〜2.0、デフォルト: 1.0)
- 設定は localStorage に保存し、再訪問時に復元します。
- 「ためしにきく」ボタンで現在表示中の標識名を試聴できます。
- オーバーレイの外側クリック、または「とじる」ボタンで閉じます。
- 設定画面表示中はキーボード操作を無効にします。

### F-06: タップフィードバック

- タップ時にスケールアニメーションで押した感覚を伝えます。
- iOS Safari では `<input type="checkbox" switch>` を利用したハプティクスフィードバックを発動します。
- Android では `navigator.vibrate()` によるバイブレーションを発動します。

## データ構造

### signs.json

```json
[
  {
    "id": "201-A",
    "name": "十形道路交差点あり",
    "category": "警戒標識",
    "image": "data/signs/201-A.png",
    "reading": "じゅうがたどうろこうさてんあり"
  }
]
```

### カテゴリ一覧

| カテゴリ | 標識番号の範囲 | 件数 |
|---|---|---|
| 案内標識 | 100 番台 | 76 |
| 警戒標識 | 200 番台 | 27 |
| 規制標識 | 300 番台 | 70 |
| 指示標識 | 400 番台 | 16 |

補助標識 (500 番台, 31 件) は画像抽出の技術的制約により除外しています。

## 画像処理パイプライン

PDF から標識画像を抽出する際に以下の処理を行います。

1. **PDF レンダリング**: PyMuPDF で 4 倍解像度 (scale=4) でページ全体を PNG にレンダリングします。
2. **セル切り出し**: PDF 内のテキスト位置情報と罫線位置から、各標識のセル座標を特定して切り出します。
3. **白背景透過**: RGB が (240, 240, 240) 以上のピクセルを透過にします。
4. **連結成分解析**: OpenCV の connectedComponentsWithStats で不透明ピクセルの連結成分を検出します。画像中心に最も近い大きな成分を標識本体と判定し、それ以外 (はみ出した文字・罫線) を除去します。
5. **内部白色復元**: 標識本体の bounding box 内で、外周から floodfill で到達できない透過ピクセルを「内部の白」と判定して白 (255, 255, 255, 255) に復元します。

## 技術スタック

| 要素 | 技術 |
|---|---|
| マークアップ | HTML5 |
| スタイル | CSS3 (media query によるレスポンシブ対応) |
| ロジック | Vanilla JavaScript (ES2020+) |
| 読み上げ | Web Speech API (SpeechSynthesis) |
| スワイプ | Touch Events API |
| ハプティクス | `<input type="checkbox" switch>` (iOS) / Vibration API (Android) |
| 設定保存 | localStorage |
| 画像抽出 | Python (PyMuPDF, PIL, OpenCV, NumPy) |
| ホスティング | 静的サイト (GitHub Pages 等) |

## 制約事項

- ネットワーク接続が必要です (初回読み込み時)。
- 読み上げ音声は端末にインストールされた音声エンジンに依存します。
- 音声一覧の取得は非同期のため、ページ読み込み直後は音声選択肢が揃わない場合があります (voiceschanged イベントで対応済み)。
- iOS のハプティクスフィードバックは `<input type="checkbox" switch>` の非公式な挙動を利用しているため、将来の Safari 更新で動作しなくなる可能性があります。
